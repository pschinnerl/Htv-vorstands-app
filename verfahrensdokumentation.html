Übersicht aller HTV-Applikationen

Der HTV betreibt vier Web-Applikationen auf Basis von Google Firebase und GitHub Pages. Alle Systeme sind kostenlos (Firebase Spark Plan) und erfordern keinen eigenen Server.

App URL Firebase-Projekt Technologie
Schlüsselverwaltung pschinnerl.github.io/schlusselapp schluesselapp-15d55 HTML / Vanilla JS
HTV-Anlagen pschinnerl.github.io/htv-anlagen htv-anlagen React + Vite
Vorstands-App helmstedtertv.github.io/Htv-vorstands-app htv-vorstands-app React + Vite + TypeScript
Vertragsmanagement helmstedtertv.github.io/htv-vertraege htv-vertraege React + Vite
Gemeinsame Infrastruktur: Alle Apps nutzen Firebase Authentication (E-Mail/Passwort) und Cloud Firestore als Datenbank. Das Hosting erfolgt über GitHub Pages – kostenlos und ohne eigenen Server.

1.6 Sicherheitshinweise

  • Der Firebase API-Schlüssel in index.html ist durch eine HTTP-Referrer-Einschränkung auf pschinnerl.github.io beschränkt.
  • Google hat im Mai 2026 eine Sicherheitswarnung über den öffentlich sichtbaren API-Schlüssel gesendet. Der Schlüssel wurde daraufhin mit Referrer-Restriction gesichert.
  • Die App-Daten sind nicht hochsensibel (Schlüsselvergabe), aber der Zugang sollte auf bekannte Nutzer beschränkt bleiben.
  • Bei Verdacht auf Missbrauch: API-Schlüssel in Google Cloud Console neu generieren und in index.html ersetzen.

3.6 Nutzerverwaltung über die App

Admins können Nutzer direkt über den Menüpunkt Mitglieder in der App einladen – ohne Firebase Console:

  1. Als Admin einloggen → Menü → „Mitglieder"
  2. „Mitglied einladen" → Name, E-Mail, Rolle eingeben
  3. Die Person erhält eine E-Mail mit Link zum Passwort setzen
  4. Rolle kann jederzeit per Dropdown geändert werden

Was ein Dritter braucht, um die Apps zu pflegen

Minimum (nur Benutzerverwaltung und Datenpflege):

  • Zugang zum Google-Konto von P. Schinnerling – oder eigenes Konto als Mitinhaber in Firebase hinzufügen
  • Firebase Console → Projekteinstellungen → Nutzer und Berechtigungen → neue E-Mail als Editor einladen

Für Codeänderungen an Schlüsselverwaltung:

  • GitHub-Zugang: Repository schlusselapp muss übergeben oder geteilt werden
  • Keine lokale Entwicklungsumgebung nötig – index.html direkt auf GitHub bearbeiten

Für Codeänderungen an HTV-Anlagen:

  • GitHub-Zugang: Repository htv-anlagen muss übergeben oder geteilt werden
  • Mac mini mit Node.js und npm – oder eigener Rechner mit Node.js
  • Projektordner /Users/Peter/htv-anlagen/ übertragen (ZIP oder Git Clone)
  • Danach: npm install im Projektordner, dann npm run deploy zum Deployment

Für Codeänderungen an Vorstands-App:

  • GitHub-Zugang zur Organisation helmstedtertv oder Repository Htv-vorstands-app
  • Node.js + npm auf dem eigenen Rechner
  • Projektordner via git clone herunterladen
  • Danach: npm install, dann npm run deploy

SchrittAktion
1. Firebase-Mitinhaber einladen Firebase Console → Projekteinstellungen (Zahnrad) → Nutzer und Berechtigungen → Mitglied hinzufügen → E-Mail der neuen Person → Rolle: Editor
2. GitHub-Repositories übergeben Persönliche Repos (pschinnerl): GitHub → Repository Settings → Collaborators → einladen; oder Transfer an neuen Besitzer.
Org-Repos (helmstedtertv): GitHub → Organisation → Settings → Members → neue Person einladen
3. Quellcode HTV-Anlagen Ordner /Users/Peter/htv-anlagen/ als ZIP exportieren und übergeben; oder: git clone vom GitHub-Repository
4. Quellcode Vorstands-App git clone https://github.com/helmstedtertv/Htv-vorstands-app – kein separater Transfer nötig
5. Zugangsdaten App-Nutzer Liste der App-Nutzer in Firebase Authentication prüfen; für Vorstands-App: Admin-Zugang in der App übergeben
6. Google Drive Protokoll-Ordner Google Drive → Ordner „HTV Protokolle" → Freigabe → neue Person als Mitbearbeiter einladen
Wichtig: Das persönliche Google-Konto von P. Schinnerling ist derzeit der einzige Eigentümer aller Firebase-Projekte. Für eine dauerhafte Nachfolge sollte mindestens eine weitere Person als Firebase-Editor eingetragen werden.
